# MVP 二代改进观察日志

本文件用于记录分级真机测试中发现的二代改进点。当前阶段只收集现象、证据和
验收指标，不在单项测试过程中穿插修改控制算法。完成整轮模式测试后，再统一
评审、设计、实现和回归，避免连续调参掩盖不同模式之间的问题。

## 处理规则

- 新发现按 `V2-XXX` 编号追加，保留原始测试模式和复现条件；
- 区分感知、目标选择、控制、驱动和可视化问题，不把所有现象都归因于增益；
- 每项改动前先保存基线数据，改动后按相同场景回归；
- 模式2、模式3和模式4全部完成前，以下项目状态保持“已记录，暂缓实现”；
- 安全停车、目标丢失停车和最终指令唯一发布者等现有约束不得因优化而放宽。

## V2-001：画面关注点优先保持在人脸

- 状态：已记录，暂缓实现
- 首次观察：2026-07-18，模式2“纯云台自动跟踪”
- 当前行为：控制器将整个人体 bbox 的几何中心对准画面中心。人体框中心通常
  位于躯干部位，因此即使跟踪稳定，构图焦点也不一定是人脸。
- 二代目标：同一人的人脸可见时，以人脸中心或头部关键点作为云台的图像目标；
  人脸暂时不可见时平滑回退到人体上半身参考点，不能立即切换到其他人。
- 候选方案：
  1. 在已锁定的人体 bbox 内运行轻量人脸检测或人脸/姿态关键点模型；
  2. 将人脸结果与当前 `tracking_id` 绑定，输出独立的控制关注点；
  3. 采用“人脸中心 → 头肩参考点 → 人体框中心”的分级回退；
  4. 对关注点切换增加时间滞回和平滑，避免人脸短暂遮挡导致云台跳变。
- 后续需要记录：人脸检测率、遮挡恢复时间、关注点跳变量、多人场景 ID 是否稳定、
  Jetson 推理负载和总帧率。
- 验收指标：整轮测试结束后根据基线数据确定；至少包含人脸可见时的构图误差、
  遮挡时不跳人以及目标丢失后的安全停车。

## V2-002：提高云台响应灵敏度

- 状态：已记录，暂缓实现
- 首次观察：2026-07-18，模式2“纯云台自动跟踪”
- 当前行为：图像误差滤波、指令滤波、较小的 P 增益和较低的最大角速度共同保证
  了低速安全，但使目标开始移动、改变方向或接近中心时的响应偏慢。
- 当前相关参数：
  - `K_gimbal_x: 0.25`
  - `K_gimbal_y: 0.18`
  - `max_gimbal_yaw_vel: 0.10 rad/s`
  - `max_gimbal_pitch_vel: 0.08 rad/s`
  - `filter_alpha_error: 0.25`
  - `filter_alpha_cmd: 0.20`
- 二代目标：缩短从目标运动到云台有效响应的端到端延迟，同时保持中心附近不抖动、
  不明显过冲、方向反转不振荡，且目标丢失仍在既定时间内停车。
- 候选方案：
  1. 分别测量相机、检测跟踪、控制发布和执行器响应延迟，先定位主要瓶颈；
  2. 在低速安全边界内逐级提高 P 增益和最大角速度；
  3. 重新评估两级低通滤波，减少不必要的相位滞后；
  4. 如单纯 P 控制无法兼顾速度与稳定性，再评估速度前馈、微分阻尼或分段增益；
  5. 水平和俯仰分别标定，不共用一组主观参数。
- 后续需要记录：目标开始移动到首个非零指令的延迟、指令到云台运动的延迟、
  回中时间、最大过冲、稳态像素误差、方向反转恢复时间和丢失停车时间。
- 验收指标：完成基线采集后确定；优化不得破坏方向正确性、死区稳定性和安全停车。
